## Requirements

Remote-control bridges die silently (this session: finance, mcp-servers,
portal--3), and the user only discovers it when reaching for the phone and
finding a session unreachable. `cctrl session doctor --fix` can already repair
bridges on demand; this plan adds an opt-in scheduled task that runs the repair
automatically so dead bridges self-heal between glances.

**Acceptance criteria:**

- [ ] A command (e.g. `cctrl session autoheal`) runs one heal pass: detect
      `dead` bridges (per existing rc-state logic) and repair them via the same
      path as `doctor --fix`, logging what it healed.
- [ ] An install/enable command sets up a recurring local timer (launchd agent
      on macOS) that runs the heal pass on an interval, and a disable command
      removes it. Installation is opt-in, never automatic.
- [ ] The heal pass is safe to run concurrently with active sessions: it only
      repairs `dead` bridges, never touches `live` ones, and is idempotent.
- [ ] **Never clobbers live work.** Before injecting `/rc` (which begins with
      `C-u`, clearing the input line), autoheal verifies the input line is empty
      (reuse plan 016's unsent-draft detection) and the session is neither `busy`
      nor in copy-mode; it skips and logs any session that fails these gates.
- [ ] Runs fully non-interactively (launchd has no TTY): uses the `--yes`
      /assume-yes path, never blocks on a prompt.
- [ ] A `--dry-run` mode reports what would be healed without acting.
- [ ] Heal actions are logged to a predictable location for after-the-fact
      inspection.

## Design

Factor the bridge-repair core out of `_session_doctor --fix` (or call it) into a
`_session_autoheal` that filters to dead bridges and repairs them, reusing the
copy-mode-guarded keystroke path. **Safety gate before any repair:** skip the
session unless it is not `busy`, not in copy-mode, AND has an empty input line
(reuse plan 016's unsent-draft detection) — the `/rc` injection starts with
`C-u`, so repairing a session with a typed-but-unsent draft would erase it. Run
non-interactively via the assume-yes path (no TTY under launchd). Add an
`autoheal` action to `cmd_session` dispatch with `--dry-run`. Add `autoheal install|uninstall` that writes/removes
a launchd plist under `~/Library/LaunchAgents/` invoking `cctrl session
autoheal` on an interval, mirroring the existing local launchd agent pattern the
user already runs (`com.matthew.proxy-orphan-guard`). Log to a file under the
cctrl data dir.

**Files expected to change:**

- `cctrl`: add `_session_autoheal`; add `autoheal` (+`install`/`uninstall`/
  `--dry-run`) to `cmd_session` dispatch + help.
- A launchd plist template (inline-generated by the install command).
- `tests/run-tests.sh`: assert the dry-run pass selects only dead bridges (via
  stubbed rc-state) and acts on none; assert `install` writes a plist to a
  test-overridable dir and `uninstall` removes it.

**Testing approach:** unit-only — bridge repair and launchctl are stubbed; no
real bridges or system agents touched in tests.

**Out of scope:** healing across hosts (local only here), and changing on-demand
`doctor --fix` behavior beyond extracting the reusable repair core.
